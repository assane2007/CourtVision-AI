   821→## 4. FAIBLESSES MAJEURES (AVANT CORRECTIONS)
   822→===============================================================================
   823→
   824→### 🔴 CRIT-1: Vulnérabilité SSRF via Caddyfile [CORRIGÉ]
   825→- `XTransformPort` permettait de proxy vers n'importe quel port interne (SSH:22, DB, etc.)
   826→- **Impact**: Accès non autorisé à tous les services internes
   827→- **Fix**: Allowlist de ports autorisés (3000, 3003 uniquement)
   828→
   829→### 🔴 CRIT-2: Secret JWT prévisible [CORRIGÉ]
   830→- `NEXTAUTH_SECRET=courtvision-ai-secret-2024-secure-key` — devinable
   831→- **Impact**: Forge de tokens JWT pour n'importe quel utilisateur
   832→- **Fix**: Secret aléatoire 64 caractères généré par openssl
   833→
   834→### 🟠 HIGH-1: Absence de rate limiting sur 12 endpoints [CORRIGÉ]
   835→- Seuls signup et login étaient rate-limited
   836→- L'endpoint IA (coûteux) n'avait aucune limite
   837→- **Fix**: Rate limiting sur tous les endpoints mutants (5-20 req/15min)
   838→
   839→### 🟠 HIGH-2: Absence de headers de sécurité [CORRIGÉ]
   840→- Pas de CSP, X-Frame-Options, X-Content-Type-Options
   841→- **Fix**: CSP + 3 headers de sécurité dans next.config.ts
   842→
   843→### 🟠 HIGH-3: Fuite mémoire MediaPipe [CORRIGÉ]
   844→- PoseLandmarker jamais disposé — fuite GPU/WASM à chaque session caméra
   845→- **Fix**: `poseLandmarkerRef.current?.close?.()` dans le cleanup
   846→
   847→### 🟠 HIGH-4: Composant monolithique 2000+ lignes [NON CORRIGÉ]
   848→- `camera-workout.tsx` contient 18+ useState, 20+ useRef, 10+ useEffect
   849→- Impossible à tester, memoïser, ou maintenir à long terme
   850→- **Recommandation**: Décomposer en hooks (useMediaPipe, useCamera, useWorkoutTimer, useRepDetection, useAIFormCheck) et sous-composants (PoseCanvas, ScoreDisplay, ControlBar, RestTimer, CountdownOverlay)
   851→
   852→### 🟠 HIGH-5: Zéro test [NON CORRIGÉ]
   853→- Aucun fichier .test.ts ou .spec.ts dans le projet
   854→- Pas de filet de sécurité pour le scoring, la détection de reps, les API
